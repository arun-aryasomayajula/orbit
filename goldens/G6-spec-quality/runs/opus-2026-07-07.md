## The core decisions in the spec
- **Math contract — flat cross-classification + explicit interaction.** Decompose over the **product partition** of the N segments (each as In/Out or its buckets → disjoint+exhaustive *by construction*, so overlap is a non-issue). This reuses Tier-1 math verbatim and keeps `total = rate + mix` exact. **Rate is reported once** (it's within-cell, not attributable). **Mix is attributed per segment** as each segment's marginal Tier-1 mix, with `interaction_pp = mix − Σ segment_mix` as a *named residual* → additive at both levels, symmetric/order-free (not hierarchical). Rejected Shapley (2^N cost, hides the interaction that *is* the insight). **N capped at 2** (product-cell explosion; matches existing caps).
- **Druid support — per-cell funnel fan-out + total-subtraction residual**, reusing the `SegmentDrilldown` primitive and the `analyze/v2` bounded-fan-out harness; math is engine-agnostic, only the row-fetch differs.
- **Degradation — per-lane, per-segment, soft.** If any selected segment isn't Druid-expressible (behavioral/cohort/regex-no-druid-binding), the Druid lane returns a structured `{degraded}` (not null, not 500) while Trino still computes fully. Expressibility is decided *per lane* (regex is Druid-ok but needs the Trino membership-CTE path).
- **API/response** — extend `mix_decomposition_v2` in place (`trino_segment_ids` list, single-int kept for back-compat); response is an additive superset of Tier 1 (`segment_effects[]` + `interaction_pp`).

## Top 3 unknowns that most need a human decision
1. **Marginal + explicit interaction vs Shapley fair-attribution** (§3.3, U1). This *blocks the response schema*. My recommendation is marginals-now / Shapley-later, but it's a genuine product+math judgment call about whether analysts want a clean N-way split or want to see the interaction term explicitly.
2. **Druid↔Trino comparability under cross-classification** (§4.4, U3). The existing event-scope (Druid) vs entity-scope (Trino cohort) semantic gap *compounds* when you cross two segments — the Compare tab would show two internally-consistent but mutually-incomparable decompositions. Label it, or suppress Compare for this mode? Product call.
3. **`cohort_axis` back-compat** (§7). Tier 1 emitted it as a string; Tier 2 wants a list — a breaking read for the current UI. Keep-string-add-list (my rec) vs migrate the reader — cheap but breaks silently if gotten wrong.
