## Emitted SQL shape
- Numeric: `Rule(FieldRef("v","amt"), "between", [10, 100])` → `("amt" >= 10 AND "amt" <= 100)` (bounds unquoted).
- String: `Rule(FieldRef("v","code"), "between", ["a", "z'z"])` → `("code" >= 'a' AND "code" <= 'z''z')` (quoted + `'`-doubled via the adapter).
- Identifier `col_expr` still comes from `resolve_field` + `adapter.quote_identifier` (bare-identifier validation + quoting unchanged); each rule is parenthesized so same-field OR-grouping stays unambiguous.
- Malformed (`1`, `"10"`, `[10]`, `[10,20,30]`, `[None,5]`, …) → `PredicateAstError` / `PredicateCompileError`, filter never partially emitted.
